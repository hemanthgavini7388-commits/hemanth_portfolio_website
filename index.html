Architecture Overview

The pipeline utilises a producer-consumer architecture — a Python-based producer continuously ingests raw sales records at 20 records/second into a MySQL staging database. A parallel consumer process applies business logic, aggregations and KPI calculations before writing to a clean reporting layer. Power BI connects via Direct Query to deliver live dashboards without scheduled refresh dependency.

Technical Stack
  • Python - Pandas for transformation, threading for producer-consumer concurrency
  • MySQL - Staging and reporting schema with optimised indexing for query performance
  • Power BI - Direct Query mode, DAX measures for KPI calculations, dynamic filters
  • ETL Design - Automated daily KPI aggregations covering revenue, orders, ratings and regional performance
SQL Sample — Daily KPI Aggregation
SELECT DATE(order_date) AS report_date, region, COUNT(order_id) AS total_orders, ROUND(SUM(revenue), 2) AS total_revenue, ROUND(AVG(customer_rating), 2) AS avg_rating, ROUND(SUM(revenue) / COUNT(order_id), 2) AS avg_order_value, ROW_NUMBER() OVER (PARTITION BY region ORDER BY SUM(revenue) DESC) AS rank FROM sales_staging WHERE status = 'completed' GROUP BY DATE(order_date), region ORDER BY report_date DESC;
Outcome

Eliminated manual daily reporting through full automation. Reporting efficiency improved by 40%. Business teams gained access to live performance tracking, enabling faster operational responses to demand fluctuations, regional performance gaps and rating deterioration.

Methodology

Analysed a decade of national healthcare expenditure data spanning multiple categories — primary care, hospitalisation, preventive programs and administrative overhead. Applied time series decomposition to separate trend, seasonal and cyclical components before building a regression-based forecasting model using scikit-learn.

Scenario modelling dashboards in Power BI and Tableau allowed stakeholders to simulate the financial impact of reallocating budget toward preventive health programs — quantifying a potential 20% cost reduction over a 5-year horizon.

Technical Stack
  • Python — scikit-learn (LinearRegression, RandomForest), Pandas, Matplotlib, Seaborn
  • SQL — Data extraction, CTEs, multi-year aggregations, YoY variance calculations
  • Power BI & Tableau — Scenario simulation dashboards, forecast visualisation, budget allocation modelling
Python Model Approach
